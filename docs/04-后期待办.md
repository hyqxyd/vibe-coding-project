# 课堂 Vibe Coding 平台后期待办

## 1. 产品能力待办

- 技术栈白名单切换
- 干预组 / 限制组能力
- 多人协作
- 跨课堂分析
- 高级教师干预
- 更丰富的教学模板库

## 2. 平台能力待办

- 归档库与在线库联动查询
- Kubernetes 化迁移
- 可插拔 Agent 编排
- 更细粒度资源调度
- 公告区发布工作流
- 限时访问结果页续期机制

## 3. 教学分析待办

- 自动评分策略优化
- 课程级 Rubric 模板库
- 认知过程可视化增强
- 学生成长轨迹分析
- 个性化推荐策略优化

## 4. 触发条件建议

以下情况出现后，建议优先启动对应待办：

- 课堂规模持续提升到多班并发：优先推进 Kubernetes 化与资源调度增强
- 教师开始区分不同教学班型：优先推进干预组 / 限制组能力
- 历史数据显著增长：优先推进归档联查和冷存储检索
- 教师评分负担明显上升：优先推进自动评分与 Rubric 模板库
- 学生优秀作品增多：优先推进公告发布工作流和访问治理
