# 课堂 Vibe Coding 平台前端页面原型草图说明

## 1. 文档目标

本说明用于给前端页面原型设计提供结构化文字草图，帮助：

- 产品快速出低保真原型
- 前端明确页面分区与交互重点
- 研发和教师端 / 学生端需求对齐

## 2. 页面范围

首期建议优先出以下原型：

- 学生工作台
- 教师课程总览页
- 教师学生详情页
- 教师评分页
- 公告展示页

## 3. 学生工作台原型说明

## 3.1 布局建议

采用“三栏 + 底部时间线”：

- 左栏：问题上下文与配置面板
- 中栏：对话区与流程图区
- 右栏：预览与日志
- 底部：Patch / 事件时间线

## 3.2 关键区域

### 左栏

- 当前课程 / 主题 / 问题信息
- 教师约束说明
- 配置面板折叠组

### 中栏

- 对话输入框
- 对话消息流
- 当前流水线节点状态图

### 右栏

- 运行预览页
- 错误日志摘要
- 修复建议区

### 底部

- 当前 Patch 摘要
- 最近事件列表
- 快照切换入口

## 4. 教师课程总览页原型说明

## 4.1 布局建议

采用“顶部筛选 + 指标卡 + 图表区”：

- 顶部：课程 / 班级 / 主题 / 问题切换
- 第一屏：6 个核心指标卡
- 第二屏：4 个分析图表
- 右侧或底部：风险项目快捷入口

## 4.2 指标卡建议

- 当前学生数
- 已提交人数
- 平均过程分
- 平均结果分
- 平均失败次数
- 当前资源占用

## 5. 教师学生详情页原型说明

## 5.1 布局建议

采用“左信息、中过程、右结果、底评分”布局。

### 左信息区

- 学生头像 / 姓名 / 学号
- 所属主题与问题
- 项目标题
- 当前状态

### 中过程区

- 认知轨迹时间线
- 对话摘要节点
- Patch 摘要节点
- 运行失败 / 修复节点

### 右结果区

- 最终预览
- 流水线视图
- 截图预览
- 公告状态

### 底评分区

- 过程分表单
- 结果分表单
- 自动评分建议卡片
- 教师评语输入框

## 6. 教师评分页原型说明

## 6.1 布局建议

采用“左列表、右评分详情”：

- 左侧：待评分项目列表
- 右侧：当前项目评分面板

## 6.2 左侧列表字段

- 学生姓名
- 项目标题
- 自动评分状态
- 教师评分状态

## 6.3 右侧面板内容

- Rubric 权重
- 分项维度
- 自动评分建议
- 分差提示
- 提交按钮

## 7. 公告展示页原型说明

## 7.1 教师发布视图

- 候选项目列表
- 发布时间设置抽屉
- 有效期设置
- 公开字段勾选

## 7.2 公共展示视图

- 作品标题
- 摘要
- 结果页入口
- 流水线视图入口
- 有效期提示

## 8. 状态与空态建议

所有页面都建议有：

- skeleton 加载态
- 无数据空态
- 权限错误态
- 超时重试态

## 9. 交互重点建议

- 学生工作台的对话与配置要双向同步
- 教师详情页中过程与结果要能联动滚动
- 评分页中自动评分建议要能一键带入
- 公告页要明显显示公开范围限制

## 10. 建议下一步

基于本说明，下一步最适合继续补充：

- 低保真线框图
- 页面组件清单
- 页面到接口映射表
- 页面验收标准
