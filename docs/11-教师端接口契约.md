# 课堂 Vibe Coding 平台教师端接口契约

## 1. 文档目标

本文件定义教师端核心接口的职责、请求参数、响应结构和状态约定，用于支撑：

- 课程总览
- 课堂管理
- 学生项目列表与详情
- 评分中心
- 公告发布

首期约定：

- 接口风格采用 REST 为主
- 响应格式统一
- 列表接口支持分页
- 详情接口返回教师端页面所需聚合数据

## 2. 通用响应结构

```json
{
  "code": 0,
  "message": "ok",
  "data": {},
  "requestId": "req_20260330_001"
}
```

### 通用字段说明

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| code | number | 业务状态码，0 表示成功 |
| message | string | 响应消息 |
| data | any | 业务数据 |
| requestId | string | 请求追踪 id |

## 3. 认证上下文

教师端接口建议统一透出：

- tenantId
- teacherId
- courseId
- classId

其中：

- `courseId` 和 `classId` 可以通过路径或 query 显式传入
- `teacherId` 由认证上下文注入

## 4. 课程总览接口

## 4.1 获取课程总览

`GET /api/teacher/courses/{courseId}/classes/{classId}/overview`

### query 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| sessionId | string | 否 | 指定课堂会话 |
| topicId | string | 否 | 指定主题 |
| questionId | string | 否 | 指定问题 |

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "summary": {
      "studentCount": 100,
      "submittedCount": 68,
      "avgProcessScore": 81.2,
      "avgResultScore": 84.5,
      "avgFailedRuns": 2.4,
      "resourceUsageRate": 0.72
    },
    "charts": {
      "stageHeatmap": [],
      "errorRanking": [],
      "aiDependencyDistribution": [],
      "scoreCompareScatter": []
    }
  }
}
```

## 5. 课堂管理接口

## 5.1 获取主题与问题配置

`GET /api/teacher/courses/{courseId}/classes/{classId}/config`

### 响应内容

- 主题列表
- 问题列表
- 当前课堂约束
- 当前资源配额

## 5.2 更新课堂约束

`PUT /api/teacher/courses/{courseId}/classes/{classId}/constraints`

### 请求示例

```json
{
  "allowFreeExploration": true,
  "publicShowEnabled": true,
  "techStackProfile": "webapp_react_fastapi_standard"
}
```

## 5.3 更新课堂资源配额

`PUT /api/teacher/courses/{courseId}/classes/{classId}/resource-quota`

### 请求示例

```json
{
  "cpu": "1",
  "memory": "1Gi",
  "ttlMinutes": 90,
  "warmPoolSize": 30
}
```

## 7.2 获取原始对话

`GET /api/teacher/projects/{projectId}/conversations`

### query 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| cursor | string | 否 | 游标 |
| limit | number | 否 | 拉取条数 |

### 返回内容

- 原始对话消息
- 角色
- 时间戳
- 相关阶段标签

## 7.3 获取运行日志摘要

`GET /api/teacher/projects/{projectId}/run-logs/summary`

### 返回内容

- 最近错误摘要
- 运行成功 / 失败次数
- 关键报错标签

## 8. 评分中心接口

## 8.1 获取 Rubric 列表

`GET /api/teacher/courses/{courseId}/rubrics`

## 8.2 创建 Rubric

`POST /api/teacher/courses/{courseId}/rubrics`

### 请求示例

```json
{
  "name": "Web 创意原型基础 Rubric",
  "weights": {
    "process": 0.4,
    "result": 0.6
  },
  "dimensions": [
    {
      "id": "problem_understanding",
      "group": "process",
      "weight": 0.15
    }
  ]
}
```

## 8.3 更新 Rubric

`PUT /api/teacher/courses/{courseId}/rubrics/{rubricId}`

## 8.4 获取自动评分建议

`GET /api/teacher/projects/{projectId}/auto-score`

### 返回内容

- 过程分建议
- 结果分建议
- 证据摘要
- 风险标记
- 生成时间

## 8.5 提交教师评分

`POST /api/teacher/projects/{projectId}/teacher-score`

### 请求示例

```json
{
  "processScore": 84,
  "resultScore": 88,
  "dimensions": [
    {
      "id": "debug_recovery",
      "score": 17,
      "comment": "能根据日志完成有效修复"
    }
  ],
  "comment": "整体完成度较好，过程较清晰"
}
```

### 响应内容

- 教师评分结果
- 与自动评分差异摘要

## 9. 公告展示接口

## 9.1 获取公告候选项目

`GET /api/teacher/courses/{courseId}/showcase/candidates`

### query 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| minScore | number | 否 | 最低教师分筛选 |
| published | boolean | 否 | 是否已发布 |

## 9.2 发布优秀作品

`POST /api/teacher/projects/{projectId}/showcase`

### 请求示例

```json
{
  "published": true,
  "publicFields": [
    "pipeline_view",
    "result_page"
  ],
  "expiresAt": "2026-04-06T00:00:00Z",
  "title": "优秀作品：校园二手书交换站",
  "summary": "完成了基础页面、详情页和留言能力"
}
```

## 9.3 更新优秀作品发布信息

`PUT /api/teacher/projects/{projectId}/showcase`

## 9.4 获取已发布公告列表

`GET /api/teacher/courses/{courseId}/showcase`

## 10. 自动评分任务接口

## 10.1 触发重算自动评分

`POST /api/teacher/projects/{projectId}/auto-score/recalculate`

## 10.2 获取自动评分任务状态

`GET /api/teacher/projects/{projectId}/auto-score/task`

### 响应示例

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "taskId": "task_score_001",
    "status": "scoring_result",
    "retryCount": 1,
    "createdAt": "2026-03-30T10:50:00Z"
  }
}
```

## 11. 错误码建议

| 错误码 | 含义 |
| --- | --- |
| 400100 | 参数错误 |
| 400404 | 项目不存在 |
| 400409 | Rubric 配置冲突 |
| 400422 | 发布字段不在允许范围内 |
| 500101 | 自动评分任务创建失败 |
| 500102 | 自动评分结果暂不可用 |

## 12. 分页与排序约定

列表接口建议统一支持：

- `page`
- `pageSize`
- `sortBy`
- `sortOrder`

排序字段示例：

- updatedAt
- processScore
- resultScore
- submittedAt

## 13. 版本与扩展约定

首期建议：

- 路径中不加版本号，由网关统一管理
- 响应字段保持前向兼容
- 后续新增字段尽量追加，不轻易改名
