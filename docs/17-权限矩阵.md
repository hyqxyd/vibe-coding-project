# 课堂 Vibe Coding 平台权限矩阵

## 1. 文档目标

本矩阵用于定义教师、学生、管理员三类核心角色在首期平台中的功能权限边界，支撑：

- 菜单显示控制
- 接口访问控制
- 页面操作开关
- 审计与权限扩展

## 2. 角色定义

| 角色 | 说明 |
| --- | --- |
| 学生 | 参与课堂创作、运行、提交与查看个人评分 |
| 教师 | 管理课程与课堂、查看学生过程、评分与发布公告 |
| 管理员 | 管理平台配置、资源池、全局策略与审计 |

## 3. 权限分级

建议统一使用四类权限动作：

- view：查看
- create：创建
- update：修改
- manage：管理或审批

## 11. 字段级权限建议

## 11.1 学生可编辑字段

- project.title
- project.description
- ui.*
- logic.*
- data.*
- api.*
- submission.showcase.publicFields 仅在教师授权流程外不开放直接修改

## 11.2 教师可编辑字段

- context.teacherConstraints.*
- runtime.resourceQuota.*
- submission.teacherScore.*
- submission.showcase.*

## 11.3 管理员可编辑字段

- runtime.profile
- 全局镜像与资源池配置
- 模型与日志治理策略

## 12. 接口鉴权建议

### 学生接口

- 仅允许访问本人 projectId
- 仅允许查看本人评分与结果

### 教师接口

- 必须绑定到自己可管理的 courseId / classId
- 不允许访问无关课程班级的学生过程数据

### 管理员接口

- 拥有全局管理能力
- 高风险操作需审计记录

## 13. 特殊约束

### 13.1 原始对话查看

- 首期教师允许查看班级学生原始对话
- 公告区不允许公开原始对话

### 13.2 运行环境控制

- 学生不能直接修改资源配额
- 教师可调整课堂资源配额
- 管理员可调整全局资源池

### 13.3 公告公开范围

- 教师只能公开流水线视图与结果页
- 学生不能自行将项目发布为公告

## 14. 权限扩展预留

后续可扩展：

- 助教角色
- 干预组 / 限制组
- 课程协同教师
- 公告审核人

## 15. 审计建议

以下操作建议必须审计：

- 教师提交评分
- 教师发布 / 下架公告
- 教师修改课堂资源配额
- 管理员修改平台全局策略
