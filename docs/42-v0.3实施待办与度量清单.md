## 3. 指标清单

## 3.1 协同效率

- 合并窗口平均阻塞时长
- PR 首轮通过率
- 冲突平均处理时长

## 3.2 质量稳定性

- AI 复核 reject 后缺陷逃逸率
- 测试补充率（源码改动伴随测试改动比例）
- 回滚率

## 3.3 记忆有效性

- 核心改动记忆覆盖率
- TEAM_INTENT 更新及时率
- 相同问题重复修复次数

## 4. 周节奏建议

- 周一：更新路由与规则
- 周三：检查中期指标
- 周五：生成周复盘并输出下周行动

## 5. 风险与应对

- 风险：规则过严导致开发变慢  
  应对：先 warning 模式运行一周，再切阻断模式

- 风险：记忆更新流于形式  
  应对：在 PR 模板中增加“随机抽查”字段

- 风险：AI 角色职责重叠  
  应对：强制 Builder / Reviewer / Test Engineer 分离
