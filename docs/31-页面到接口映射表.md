# 课堂 Vibe Coding 平台页面到接口映射表

## 1. 文档目标

本文件用于定义前端页面与后端接口之间的对应关系，帮助：

- 前端明确每个页面依赖哪些接口
- 后端明确接口服务于哪些页面
- 联调时快速定位页面缺口

## 2. 页面范围

首期覆盖：

- 学生工作台
- 教师课程总览页
- 教师项目列表页
- 教师学生详情页
- 教师评分页
- 教师公告页

## 3. 学生工作台映射

| 页面区域 | 接口 | 用途 |
| --- | --- | --- |
| 课程 / 问题上下文 | GET /api/student/courses/{courseId}/classes/{classId}/config | 拉取主题、问题、约束 |
| 项目初始化 | POST /student/projects | 创建项目 |
| Schema 获取 | GET /student/projects/{projectId}/schema | 拉取当前 Schema |
| Patch 提交 | POST /student/projects/{projectId}/patches | 提交配置 / 对话变更 |
| 运行项目 | POST /student/projects/{projectId}/run | 启动运行 |
| 查看日志 | GET /student/projects/{projectId}/run-logs | 查看运行日志 |
| 提交项目 | POST /student/projects/{projectId}/submit | 提交结果 |
| 查看评分 | GET /student/projects/{projectId}/scores | 查看教师分与自动评分 |

## 9. 页面与接口依赖优先级

| 页面 | 必须接口 | 可延后接口 |
| --- | --- | --- |
| 学生工作台 | 项目创建、Schema 获取、Patch 提交、运行、提交 | 评分查看 |
| 教师课程总览页 | overview | 高级图表增强 |
| 教师项目列表页 | projects | 批量操作增强 |
| 教师学生详情页 | project detail、conversations、run-logs、teacher-score | 高级回放 |
| 教师评分页 | rubrics、teacher-score、auto-score | 自动重算增强 |
| 教师公告页 | showcase candidates、showcase publish | 续期治理增强 |

## 10. 联调建议

- 页面联调优先按“先列表、后详情、再写操作”进行
- 同一页面涉及多个接口时，先验证聚合详情接口，再验证子接口
- 学生工作台优先打通 Patch、运行、提交三条接口
