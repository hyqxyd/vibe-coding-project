# 课堂 Vibe Coding 平台角色到菜单映射表

## 1. 文档目标

本文件用于定义不同角色登录后应看到哪些菜单、哪些菜单只读、哪些菜单可操作，支撑：

- 前端导航渲染
- 角色化首页
- 权限显隐控制
- 后续助教与干预组扩展

## 2. 角色范围

首期覆盖：

- 学生
- 教师
- 管理员

后续预留：

- 助教
- 干预组
- 公告审核人

## 3. 菜单设计原则

- 只展示角色有意义的菜单
- 不把无权限菜单展示为可点击死链
- 高风险管理菜单只对管理员开放
- 同一菜单可有“可见但只读”状态

## 7. 首页落地建议

## 7.1 学生首页

优先展示：

- 当前课程与任务
- 最近项目
- 继续创作入口
- 个人成绩入口
- 公告区入口

## 7.2 教师首页

优先展示：

- 课程总览指标卡
- 待评分数量
- 风险项目数量
- 公告候选数量

## 7.3 管理员首页

优先展示：

- 资源池状态
- 课堂并发概览
- 异常告警
- 审计概览

## 8. 菜单显隐规则

### 学生

- 不显示教师端与管理员端菜单
- 不显示公告发布入口
- 不显示 Rubric 配置入口

### 教师

- 不显示学生工作台菜单
- 不显示管理员治理菜单
- 可看到公告发布入口，但不可修改全局公告治理策略

### 管理员

- 默认不进入学生或教师具体操作页
- 可通过管理入口查看对应数据，但应保持审计记录

## 9. 菜单只读规则

以下场景建议菜单可见但操作只读：

- 教师查看自动评分建议时，仅可 view
- 管理员进入课程数据时，可 view 或 manage，按场景限制
- 学生在已提交且教师锁定后，创作工作台可只读查看历史结果

## 10. 扩展角色预留

## 10.1 助教

建议菜单：

- 课程总览
- 学生项目
- 只读评分辅助页

不建议：

- 修改资源配额
- 发布公告
- 修改全局 Rubric

## 10.2 干预组

建议菜单：

- 学生项目
- 受限创作工作台
- 特殊引导面板

## 11. 前端实现建议

菜单结构建议统一配置为：

```json
{
  "role": "teacher",
  "menus": [
    {
      "key": "course_overview",
      "visible": true,
      "permission": "view"
    }
  ]
}
```

### 前端渲染建议

- 登录后先根据角色拉菜单
- 菜单配置由后端返回或前端静态表驱动
- 页面级仍需后端接口鉴权，不只依赖前端显隐
