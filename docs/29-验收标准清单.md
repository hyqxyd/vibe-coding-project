# 课堂 Vibe Coding 平台验收标准清单

## 1. 文档目标

本清单用于定义 MVP 阶段的核心验收标准，帮助团队在联调、测试和上线前对“什么算完成”形成统一口径。

## 2. 验收原则

- 以主流程闭环为优先
- 以用户可感知结果为标准
- 以课堂场景可用性为目标
- 以教师与学生双侧体验共同验收

## 3. 学生端验收标准

## 3.1 进入与创作

- 学生可以进入所属课程与课堂
- 学生可以查看主题、问题和课堂约束
- 学生可以创建个人项目
- 学生可以使用对话和配置面板两种方式修改项目

## 3.2 生成与运行

- 配置或对话修改后能生成有效 Patch
- 前端与后端代码能按 Schema 联动生成
- 学生可以正常点击运行
- 预览和日志可以正常查看

## 3.3 提交与查看

- 学生可以提交结果
- 学生可以查看自己的评分结果
- 学生可以查看系统自动评分建议摘要

## 4. 教师端验收标准

## 4.1 课堂管理

- 教师可以创建 / 管理课堂、主题、问题
- 教师可以修改课堂约束与资源配额

## 4.2 观察与评分

- 教师可以查看项目列表
- 教师可以查看学生详情
- 教师可以查看认知时间线、日志摘要、结果预览
- 教师可以配置 Rubric
- 教师可以提交教师评分

## 4.3 公告展示

- 教师可以查看候选项目
- 教师可以发布优秀作品
- 公告只公开流水线视图和结果页
- 公告结果页支持限时访问

## 5. 平台能力验收标准

## 5.1 Schema 与 Patch

- Schema 快照可正常保存
- Patch 可正常应用
- Patch 冲突有明确提示
- 联动校验可输出诊断结果

## 5.2 代码生成

- 前端生成器可以生成页面、组件、路由
- 后端生成器可以生成接口、模型、服务
- 联动校验器可以发现前后端不一致
- 修复生成器失败时有明确提示

## 5.3 运行调度

- Docker 容器池可正常分配
- 运行环境能启动与释放
- 运行日志可采集
- 预览地址可访问

## 5.4 自动评分与认知分析

- 提交后可以进入自动评分任务
- 自动评分建议可在合理时间内生成
- 教师端可以看到基础认知分析图表

## 6. 权限验收标准

- 学生不能访问教师端接口
- 教师不能访问无关课程班级的数据
- 管理员可以管理全局配置
- 公告区不公开原始对话与评分细节

## 7. 状态码与错误提示验收标准

- 关键错误场景均有稳定状态码
- 前端提示文案与等级符合设计
- 可重试场景有明确“稍后重试”提示
- 高风险错误展示 requestId

## 8. 公告与访问验收标准

- 教师发布后公告列表即时可见
- 限时结果页在有效期内可访问
- 过期后返回明确提示
- 公开字段仅限允许范围

## 9. 性能与稳定性验收标准

- 100 人课堂级场景可正常进入系统
- 主要创作流程无大面积阻塞
- 高峰期运行请求具备排队或重试机制
- 自动评分不阻塞课堂主链路

## 9.1 建议量化阈值

为避免“可用但不可量化”的验收争议，建议上线前至少按下列阈值验收：

- 登录 / 配置读取 / 项目创建 / Patch / 提交成功率不低于 99%
- 首屏配置读取 P95 不高于 3 秒
- 项目创建与 Patch 提交 P95 不高于 2 秒
- 项目提交 P95 不高于 3 秒
- 教师项目详情聚合接口 P95 不高于 5 秒
- 运行请求进入排队或返回受理结果 P95 不高于 3 秒
- 预热命中场景下运行启动到可访问 P95 不高于 15 秒

## 10. 测试通过口径

建议至少满足：

- P0 主流程测试全部通过
- 教师端联调用例全部通过
- 关键权限回归通过
- 公告访问场景通过
- 自动评分任务流通过

## 11. 上线前阻断项

以下任一项未通过，建议阻断上线：

- 学生无法稳定提交项目
- 教师无法完成评分闭环
- 公告结果页无法正常访问
- 权限出现越权问题
- 100 人课堂场景出现明显不可用
