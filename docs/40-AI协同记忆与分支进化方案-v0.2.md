# 课堂 Vibe Coding 平台 AI 协同记忆与分支进化方案 v0.2

## 1. v0.2 升级目标

在 v0.1 基础上，v0.2 新增两项硬能力：

- 自动化记忆门禁检查
- 冲突热点预警与团队意图快照

## 2. 新增能力清单

## 2.1 自动化脚本

- `scripts/ai_collab/memory_guard.py`
  - 检查核心改动是否同步更新协同记忆文件
  - 命中热点文件时触发阻断或预警
- `scripts/ai_collab/build_team_intent.py`
  - 汇总分支、文件、AI会话、同步事件
  - 生成 `/.ai-collab/TEAM_INTENT.md`

## 2.2 配置文件

- `/.ai-collab/GUARD_RULES.json`
  - 核心改动范围
  - 必需记忆文件
  - 热点阻断默认策略

## 2.3 v0.2.1 新增脚本

- `scripts/ai_collab/ai_test_engineer_check.py`
  - 检查源码改动是否同步补充测试变更
  - 检查接口契约与页面映射文档是否同步更新
- `scripts/ai_collab/build_weekly_review.py`
  - 自动聚合 CHANGELOG、TEAM_INTENT、FILE_MEMORY、AGENT_MEMORY
  - 生成 `/.ai-collab/WEEKLY_REVIEW.md`

## 3. 执行协议

## 3.1 提交前门禁

```bash
python scripts/ai_collab/memory_guard.py --staged-only
```

阻断条件：

- 检测到核心改动但未更新协同记忆
- 检测到热点文件改动且未明确放行

## 3.2 合并后知识库刷新

```bash
python scripts/ai_collab/build_team_intent.py
```

输出文件：

- `/.ai-collab/TEAM_INTENT.md`

用途：

- 作为下一轮 AI 辅助的默认上下文
- 作为评审与测试工程角色的统一输入

## 3.3 v0.2.1 门禁扩展

```bash
python scripts/ai_collab/ai_test_engineer_check.py --staged-only
python scripts/ai_collab/build_weekly_review.py
```

新增阻断条件：

- 检测到源码改动但无测试变更
- 检测到接口契约文档与页面映射文档不同步

## 4. Harness Engineering 落地

v0.2 将 AI 协同正式拆分为四个责任角色：

- **AI Builder**：实现需求与改动
- **AI Reviewer**：对齐架构、接口、风险
- **AI Test Engineer**：补充测试、验证正确性
- **Human Owner**：最终判断是否合并

## 5. 团队知识库结构

## 5.1 事实层

- `CHANGELOG.md`
- `BRANCH_REGISTRY.md`

## 5.2 语义层

- `FILE_MEMORY.md`
- `AGENT_MEMORY.md`

## 5.3 快照层

- `TEAM_INTENT.md`

## 6. 试运行计划（v0.2）

## 第 1-3 天

- 全员接入提交门禁脚本
- 按规则更新记忆文件
- 建立首次 TEAM_INTENT 快照

## 第 4-7 天

- 引入 AI Reviewer 与 AI Test Engineer 双复核
- 统计阻断次数、冲突次数、回滚次数

## 第 8-14 天

- 调整 GUARD 规则阈值
- 输出 v0.2 复盘报告
- 规划 v0.3 自动路由与冲突预测

## 7. 验收指标

- 核心改动无记忆更新的漏报率接近 0
- 热点文件冲突率显著下降
- AI 复核命中真实风险比例提升
- PR 首轮通过率与测试通过率提升
