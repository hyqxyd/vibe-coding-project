# 课堂 Vibe Coding 平台上线验收检查清单

## 1. 文档目标

本清单用于把验收标准进一步落实为上线前可逐项勾选的检查项，帮助团队在正式开放课堂使用前完成：

- 功能核对
- 联调核对
- 数据核对
- 容量核对
- 风险阻断核对

## 2. 使用原则

- 以课堂首期 MVP 边界为准
- 以“可实际支撑 100 人课堂”作为最终目标
- 以阻断项优先于优化项
- 以教师端、学生端、平台侧三方联合确认作为通过依据

## 3. 使用方式

建议按以下顺序执行上线验收：

1. 环境与配置检查
2. 数据与权限检查
3. 学生主流程检查
4. 教师主流程检查
5. 公告与评分检查
6. 稳定性与压测检查
7. 上线阻断项复核

## 10. 上线阻断项

出现以下任一情况，建议阻断上线：

- 学生无法稳定创建、运行或提交项目
- 教师无法查看详情、评分或发布公告
- 项目存在越权访问
- 公告页错误公开原始对话或评分细节
- 运行容器无法按需分配或无法回收
- 高峰期出现持续性大面积超时
