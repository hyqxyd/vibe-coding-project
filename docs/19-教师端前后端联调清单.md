# 课堂 Vibe Coding 平台教师端前后端联调清单

## 1. 文档目标

本清单用于指导教师端在前后端联调阶段按页面与模块逐项验证，确保：

- 页面字段与接口返回一致
- 权限与错误提示符合预期
- 评分、公告、认知分析等核心流程可闭环

## 2. 联调原则

- 先核心页面，后增强页面
- 先只读查看，后写操作
- 先单项目，后批量场景
- 每项联调都记录请求、响应、前端表现和异常

## 3. 联调环境前置条件

联调前建议确认：

- 教师账号可正常登录
- 至少存在 1 门课程、1 个班级、1 个主题、1 个问题
- 至少存在 3 个学生项目，且状态覆盖 draft / submitted / published
- 自动评分有一条 ready、一条 pending、一条 failed 状态样本
- 公告区至少有一条已发布样本

## 4. 优先级

### P0

- 课程总览页
- 学生项目列表页
- 学生详情页
- 教师评分提交
- 公告发布

### P1

- Rubric 配置页
- 自动评分重算
- 课堂资源配额更新

## 5. 课程总览页联调

## 5.1 数据加载

- 进入页面是否能正常请求 overview 接口
- 指标卡数据是否完整显示
- 图表区是否能正确渲染空状态和有数据状态

## 5.2 筛选联动

- 切换主题 / 问题后是否重新拉取数据
- sessionId、topicId、questionId 是否透传正确

## 5.3 异常处理

- overview 接口报错时是否显示友好错误提示
- 无权限访问时是否拦截并跳转

## 6. 学生项目列表页联调

## 6.1 列表展示

- 分页是否正常
- 排序是否生效
- 筛选条件是否生效
- 各列字段是否与接口返回一致

## 6.2 状态展示

- runStatus 是否正确显示
- autoScoreStatus 是否正确显示
- aiDependencyLevel 是否正确映射标签颜色

## 6.3 交互跳转

- 点击项目是否进入详情页
- 项目不存在时是否处理 400400

## 7. 学生详情页联调

## 7.1 基础信息区

- student 信息、project 信息是否齐全
- submittedAt 是否格式化正确

## 7.2 认知分析区

- timeline 是否按时间排序
- metrics 是否展示正确
- flags 是否渲染为标签列表

## 7.3 结果预览区

- resultUrl 是否可打开
- screenshots 为空时是否显示空态
- pipelineView 是否能正确渲染

## 7.4 对话与日志区

- 原始对话接口是否支持分页 / 游标
- 日志摘要是否正常显示
- 无数据时是否显示空态

## 8. 评分中心联调

## 8.1 Rubric 列表

- 是否能加载课程 Rubric 列表
- 是否能区分课程默认 Rubric 与问题级配置

## 8.2 Rubric 新建 / 更新

- 表单提交是否正确
- 权重和维度校验是否生效
- 冲突错误码是否正确提示

## 8.3 自动评分建议

- auto-score 接口 ready 时是否正确展示分数与证据
- pending 时是否显示处理中
- failed 时是否显示稍后重试

## 8.4 教师评分提交

- 教师评分表单是否能提交成功
- 提交后列表与详情是否刷新
- 与自动评分差异是否可见

## 9. 公告发布联调

## 9.1 公告候选列表

- 候选项目筛选是否生效
- published 状态过滤是否正确

## 9.2 发布流程

- 只允许选择 `pipeline_view` 和 `result_page`
- expiresAt 不合法时是否提示
- 发布成功后详情页与公告页状态是否同步

## 9.3 结果访问

- 限时地址是否可访问
- 过期地址是否返回明确错误

## 10. 权限联调

## 10.1 教师权限

- 教师是否只能访问自己课程 / 班级数据
- 无权限项目是否返回 400204

## 10.2 学生越权验证

- 学生是否无法调用教师接口
- 学生是否无法发布公告

## 10.3 管理员验证

- 管理员是否能查看教师端所有课程信息

## 11. 错误码联调

建议至少覆盖以下错误场景：

- 400100 参数错误
- 400201 无权限
- 400300 Patch / Schema 异常
- 400400 项目不存在
- 400500 公告公开字段错误
- 500100 自动评分任务失败
- 500300 沙箱资源不足

## 12. 联调记录模板

建议每项联调记录：

| 字段 | 说明 |
| --- | --- |
| 页面 / 模块 | 例如学生详情页 |
| 用例名称 | 例如加载自动评分建议 |
| 请求接口 | 接口路径 |
| 输入参数 | query / body |
| 期望结果 | 页面与接口预期 |
| 实际结果 | 实际表现 |
| 问题等级 | blocker / major / minor |
| 备注 | 修复建议 |

## 13. 推荐联调顺序

1. 课程总览页只读验证
2. 学生项目列表页分页与筛选
3. 学生详情页聚合数据加载
4. 自动评分建议展示
5. 教师评分提交
6. 公告发布与结果访问
7. Rubric 配置修改
8. 权限与错误码验证

## 14. 出口标准

教师端首期联调完成建议满足：

- P0 页面全部可访问
- 核心接口全部返回稳定
- 教师评分可闭环
- 公告发布可闭环
- 自动评分建议可正常展示
- 主要错误码和权限场景已验证

- 教师端接口联调用例表
- 前端页面验收标准
- 错误码联调脚本清单
- 权限回归测试清单
