# 课堂 Vibe Coding 平台数据库表设计草案

## 1. 文档目标

本草案用于定义平台首期核心数据表的职责、主键关系、关键字段与分层边界，支撑：

- 课程与课堂管理
- 学生项目与 Schema 快照
- 对话、Patch、事件留痕
- 教师评分、自动评分与公告展示

## 2. 设计原则

- 先满足首期课堂业务闭环
- 热数据与归档数据结构兼容
- 业务主表与事件日志分离
- 主表结构稳定，分析表可后续增强

## 3. 分库分域建议

首期建议在同一数据库中按逻辑域分表，后续再拆分：

1. 组织域
2. 教学域
3. 项目域
4. 过程域
5. 评分与公告域
6. 平台治理域

## 10. 核心关系建议

```mermaid
erDiagram
    course ||--o{ class : contains
    class ||--o{ class_session : has
    class_session ||--o{ topic : has
    topic ||--o{ question : has
    class ||--o{ student_project : owns
    question ||--o{ student_project : binds
    student_project ||--o{ project_schema_snapshot : has
    student_project ||--o{ project_patch : has
    student_project ||--o{ project_event : has
    student_project ||--o{ project_conversation : has
    student_project ||--o{ project_run_record : has
    student_project ||--o{ teacher_score : has
    student_project ||--o{ auto_score_result : has
    student_project ||--o{ showcase_record : has
```

## 11. 索引建议

首期建议重点索引：

- `student_project(class_id, question_id, status)`
- `project_event(project_id, created_at)`
- `project_conversation(project_id, created_at)`
- `project_patch(project_id, created_at)`
- `teacher_score(project_id)`
- `auto_score_result(project_id, status)`
- `showcase_record(status, expires_at)`

## 12. 归档建议

考虑到对话和事件长期保留，建议：

- 主表保留近 4 个月或一个学期热数据
- `project_conversation`、`project_event`、`project_run_record` 优先进入归档策略
- 归档表结构与在线表保持一致，方便后续迁移与查询扩展

## 13. 首期不建议建得过细的表

- 组件级细粒度行为表
- 每个自动评分证据单独一张表
- 复杂权限点明细表
- 多租户跨库分片表
