# 课堂 Vibe Coding 平台 Sprint 计划表

## 1. 文档目标

本计划表用于将 MVP 路线图与研发任务拆分清单进一步落实为可执行的 Sprint 节奏，帮助团队：

- 按阶段推进研发
- 控制每个 Sprint 范围
- 对齐前后端、平台、AI、测试投入

## 2. 计划原则

- 每个 Sprint 都应形成可演示结果
- 优先保证主链路闭环
- 风险高的能力提前验证
- 分析增强类能力后置

## 3. Sprint 划分建议

首期建议分 6 个 Sprint：

1. Sprint 1：基础建模与权限骨架
2. Sprint 2：学生工作台骨架与 Schema 基础
3. Sprint 3：生成与运行闭环
4. Sprint 4：教师端观察与评分
5. Sprint 5：公告展示与自动评分
6. Sprint 6：联调、压测与上线准备

## 4. Sprint 1：基础建模与权限骨架

### 目标

打通课程、班级、课堂、主题、问题、用户、角色与权限基础。

### 核心任务

- 用户、课程、班级、课堂、主题、问题表结构
- 登录与角色鉴权
- 基础课程 / 班级接口
- 角色到菜单骨架

### 交付结果

- 课程与课堂模型可被管理
- 学生 / 教师 / 管理员登录后看到各自基础入口

### 主要风险

- 角色边界没定好导致后续返工

## 5. Sprint 2：学生工作台骨架与 Schema 基础

### 目标

完成学生工作台壳层、Schema 类型与 Patch 基础能力。

### 核心任务

- 学生工作台布局
- 对话区与配置面板壳层
- Project Schema 类型与字段校验器
- Schema 快照与 Patch 存储

### 交付结果

- 学生可以创建个人项目
- 对话和配置修改可以落到 Schema / Patch

### 主要风险

- 配置面板与 Schema 映射设计不稳定

## 6. Sprint 3：生成与运行闭环

### 目标

打通学生从配置到代码生成到运行预览的闭环。

### 核心任务

- 前端生成器 MVP
- 后端生成器 MVP
- 联动校验器 MVP
- Docker 容器池与运行调度
- 预览与日志面板

### 交付结果

- 学生可以生成轻量前后端原型
- 平台可以分配沙箱并运行预览

### 主要风险

- 运行环境启动耗时过长
- 前后端字段联动不稳定

## 7. Sprint 4：教师端观察与评分

### 目标

打通教师查看学生项目、认知过程和评分的闭环。

### 核心任务

- 教师课程总览页
- 学生项目列表与详情页
- Rubric 配置页
- 教师评分提交

### 交付结果

- 教师能看到学生的过程与结果
- 教师能完成评分闭环

### 主要风险

- 详情页聚合接口过重
- 评分维度与页面字段不一致

## 8. Sprint 5：公告展示与自动评分

### 目标

补齐优秀作品发布和系统自动评分建议。

### 核心任务

- 公告候选页与发布流程
- 限时结果页访问
- 自动评分异步任务
- 认知标签基础计算

### 交付结果

- 教师可发布优秀作品
- 提交后可生成自动评分建议

### 主要风险

- 自动评分结果质量与教师期待不一致
- 公告访问地址失效策略不清晰

## 9. Sprint 6：联调、压测与上线准备

### 目标

完成前后端联调、错误码校验、权限回归和 100 人课堂场景验证。

### 核心任务

- 教师端接口联调
- 学生端主流程联调
- 状态码与提示文案验证
- 权限回归
- 压测与资源池调优

### 交付结果

- 核心流程稳定
- 100 人课堂达到可接受体验

### 主要风险

- 资源池峰值不足
- 部分模块集成后出现接口不一致

## 11. 每个 Sprint 的出口标准

### Sprint 1

- 课程 / 班级 / 课堂 / 主题 / 问题可配置
- 角色登录与权限骨架可用

### Sprint 2

- 学生工作台可进入
- Schema / Patch 可正常落库

### Sprint 3

- 学生可运行生成结果
- 运行日志与预览可查看

### Sprint 4

- 教师可查看项目、查看详情、完成评分

### Sprint 5

- 公告展示可发布
- 自动评分建议可生成

### Sprint 6

- 核心流程联调通过
- 压测结果达到目标
