# 课堂 Vibe Coding 平台状态码到前端提示文案映射表

## 1. 文档目标

本文件用于将接口状态码统一映射为前端展示文案、提示等级和交互动作，确保：

- 学生端与教师端错误提示风格一致
- 状态码不仅可追踪，也可被用户理解
- 前端能根据错误类型采取正确交互策略

## 2. 设计原则

- 优先给用户可执行的下一步
- 避免技术细节直接暴露给普通用户
- 同一状态码在不同端可有不同提示文案
- 高风险错误必须保留 requestId 方便排查

## 3. 提示结构建议

```json
{
  "code": 400305,
  "level": "warning",
  "title": "配置未通过校验",
  "message": "当前配置存在冲突，请根据提示调整后重试。",
  "action": "show_field_errors",
  "retryable": true
}
```

## 13. 前端动作枚举建议

| 动作 | 含义 |
| --- | --- |
| inline_or_toast | 字段内或轻提示 |
| show_field_errors | 展示字段级错误 |
| reset_query | 重置筛选参数 |
| reset_sort | 重置排序 |
| redirect_login | 跳转登录 |
| block_action | 阻止继续操作 |
| back_to_list | 返回列表页 |
| force_refresh | 强制刷新页面数据 |
| open_linkage_panel | 打开联动问题面板 |
| request_renewal | 请求延长访问 |
| open_diagnostics | 打开诊断面板 |
| retry_generation | 允许重试生成 |
| queue_or_retry | 排队或稍后重试 |
| show_request_id | 显示请求编号 |
