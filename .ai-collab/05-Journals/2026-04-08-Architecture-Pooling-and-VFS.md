# Architecture: Vibe Coding 改良版架构方案

经过行业技术方案的深度对比与推敲，我们最终确立了以下 Vibe Coding 平台架构演进路线：

## 1. 容器管理：温启动池化 (Warm Pool)
**原理与方案**: 
- 放弃原有的冷启动模式。后端（Go Data Plane）常驻一组（少量）处于 Sleep 状态的 Alpine 容器。
- 学生进入界面或触发分配时，直接从池中即拿即用（秒级/毫秒级分配）。
- 容器被领用后，后台协程异步创建新的容器补充至池中，保证水位，对用户体验零影响。

## 2. 文件同步：防抖静默同步 (Debounced Sync)
**原理与方案**:
- 放弃每次按键实时同步（高并发高开销），也放弃纯手动点击全量 Deploy 的延迟感。
- 学生敲击代码停顿 1 秒后，前端触发自动同步，仅将**修改过的单文件**内容增量下发给绑定的 Docker 容器。
- 容器内持久化这些文件，既保障了互动性，又避免了最后全量打包同步的庞大 IO。

## 3. 节点解绑：DSL 驱动编排 (DSL Driven Orchestration)
**原理与方案**:
- 前端流程图中节点（Agent）的连线与解绑，本质上是工作流拓扑结构的改变。
- 连线变化只会影响一个全局的配置或入口编排文件（如 `workflow.json` 或 `main.go` / `main.py`）。
- 绝不主动删除学生的 Agent 源代码文件。解绑只是意味着在当前的工作流执行链路中不调用该节点代码，源文件依然保存在文件系统中供日后使用。
